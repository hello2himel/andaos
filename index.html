🥚 Why "AndaOS"?

In Bengali, anda literally means egg, but colloquially? It's how you call someone charmingly stupid. We embrace the simplicity. Keep it simple, keep it stupid—that's the whole philosophy right there.

The KISS Principle

Every component must answer: "Why is this here?" If not essential—removed. If optional—made optional. If configurable—made transparent.

01

Instant Productivity

XFCE desktop pre-configured with sensible defaults. Boot and start working immediately. No week-long configuration marathons.

02

Fully Auditable

Every package, every config file is documented and justified. No mysteries. No telemetry. You know exactly what's running.

03

Arch Compatible

Uses official Arch repositories. Full access to AUR and comprehensive Arch documentation. The best of both worlds.

04

Zero Bloat

Only essential packages. No hidden services, no unnecessary daemons. Fast boot, low memory footprint, responsive desktop.

05

User Control

No forced updates, no vendor lock-in, no corporate decisions. You own your system completely and make all the choices.

06

Reproducible

Build identical ISOs from source using standard ArchISO tooling. Transparent, auditable, and completely open.

What's included

Everything you need. Nothing you don't.

In the box

  • XFCE 4 desktop environment
  • Firefox with privacy-focused defaults
  • PipeWire for modern audio routing
  • NetworkManager with CLI and applet
  • Essential tools: nano, vim, git, htop
  • Full hardware support firmware
  • Bluetooth stack with GUI controls
  • Complete archive support (zip, tar, 7z)

Not included

  • Office suites (install LibreOffice if needed)
  • Development environments (add your stack)
  • Gaming platforms (Steam/Lutris available)
  • Bloatware and telemetry
  • Unnecessary background services
  • Hand-holding abstractions
  • Mystery processes consuming RAM
  • Software you'll never use
Philosophy: AndaOS gives you a foundation. You add what you need. Your machine, your rules, your choices.

Getting Started

Three simple steps to your new operating system.

1

Write ISO to USB

Download the ISO and write it to a USB drive. Use dd on Linux/macOS, or Rufus/balenaEtcher on Windows.

# Linux/macOS
sudo dd bs=4M if=andaos-*.iso of=/dev/sdX status=progress oflag=sync

# Verify integrity
sha256sum -c SHA256SUMS
2

Boot from USB

Restart your computer and boot from the USB drive. Disable Secure Boot if enabled. The system auto-boots to a live XFCE desktop—no password required.

3

Install to Disk

Follow the Arch installation guide or use manual partitioning. AndaOS is 100% compatible with standard Arch installation procedures.

# Partition, format, mount
cfdisk /dev/sda
mkfs.ext4 /dev/sda1
mount /dev/sda1 /mnt

# Install base system
pacstrap /mnt base linux xfce4 networkmanager
genfstab -U /mnt >> /mnt/etc/fstab

Build Your Own

Fork it, customize it, make it yours. That's the whole point.

# Prerequisites on Arch Linux
sudo pacman -S archiso git base-devel

# Clone the repository
git clone https://github.com/hello2himel/andaos.git
cd andaos

# Build ISO (automated script)
./build.sh

# Or manual build
sudo mkarchiso -v -w work -o out .

# Test in QEMU before deploying
qemu-system-x86_64 -enable-kvm -m 4096 -boot d -cdrom out/andaos-*.iso
Pro tip: Edit packages.x86_64 to add your favorite software, modify airootfs/etc/skel/ for custom configs, and change airootfs/etc/os-release for your own branding. It's your system—make it reflect that.

Who is this for?

AndaOS isn't for everyone. And that's perfectly fine.

Perfect for

  • Arch enthusiasts who want pre-configured convenience
  • Developers who need a distraction-free workspace
  • System administrators deploying consistent environments
  • Students learning Linux fundamentals
  • Privacy-conscious users wanting transparency
  • Tinkerers who enjoy understanding their system
  • Anyone who values simplicity over hand-holding

Not recommended for

  • Users expecting Ubuntu-style guidance
  • Those who want every package pre-installed
  • Anyone uncomfortable with the terminal
  • People who need commercial support
  • Users wanting bleeding-edge experimental features
  • Those who prefer point-and-click for everything
